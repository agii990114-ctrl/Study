# 2. form 태그

1. form : action / method
	1) action : 정보를 보내는 주소
	2) method : 정보를 보내는 방식 (get/ post)
		- get : 입력된 모든 정보를 주소창에 반영
		- post : 입력된 정보 중 제한된 정보를 제외하여 주소창에 반영 (백엔드)

2. input의 type

	- text : 기본 텍스트 - 한 줄의 텍스트를 입력받는 기본값입니다. type 속성을 생략하면 이 값이 적용됩니다.
	- password : 비밀번호 - 입력된 텍스트가 가려져서 표시됩니다 (일반적으로 * 또는 •).
	- checkbox : 체크 박스 - 여러 옵션 중 다중 선택을 허용합니다. 선택되었을 때만 서버로 값이 전송됩니다.
	- radio : 라디오 버튼 - 동일한 name을 가진 그룹 내에서 단일 선택만 허용합니다.
	- submit : 제출 버튼 - 폼의 데이터를 지정된 서버 경로로 전송하는 버튼을 만듭니다.
	- reset : 초기화 버튼 - 폼 내의 모든 입력 필드를 기본값으로 되돌리는 버튼을 만듭니다.
	- button : 일반 버튼 - 특별한 기본 동작이 없는 일반 버튼입니다. 주로 JavaScript와 연결하여 특정 기능을 수행하도록 만듭니다.
	- file : 파일 선택 - 사용자가 로컬 장치에서 파일을 선택하고 서버로 업로드할 수 있도록 합니다. accept 속성으로 허용 파일 형식을 지정할 수 있습니다.
	- hidden : 숨겨진 필드 - 사용자에게는 보이지 않지만, 폼을 제출할 때 특정 데이터를 서버로 함께 전송할 때 사용됩니다.

# 3. 웹서버 / 웹 어플리케이션 서버 (WAS)

1.웹서버 - Nginx, Apache HTTP Serverm, IIS (Internet Information Services)

: 클라이언트(보통 웹 브라우저)의 요청을 받아 HTML 문서, 이미지, CSS, JavaScript 파일과 같은 웹 콘텐츠를 전송하는 소프트웨어와, 그 소프트웨어가 설치된 하드웨어를 통칭하는 용어입니다.
쉽게 말해, 사용자가 인터넷 주소(URL)를 입력했을 때 그에 해당하는 정보를 찾아서 사용자에게 보여주는 역할을 담당하는 서버입니다.

2.웹 어플리케이션 서버 (Web Application Server) - Apache Tomcat, Jeus, WebLogic, WebSphere

: 동적인 웹 콘텐츠를 생성하고 웹 애플리케이션의 비즈니스 로직을 실행하는 데 특화된 서버입니다.
WAS는 단순한 웹 서버(Web Server)의 기능을 포함하면서도, 더 복잡하고 기능적인 역할을 수행하여 현대의 상호작용적인 웹 서비스를 가능하게 합니다.

		
# 4. doker에서의 nginx 다운방법

- doker :  개발자들이 애플리케이션을 더 쉽고 효율적으로 개발, 배포, 실행할 수 있도록 돕는 컨테이너 기반의 가상화 플랫폼입니다.
쉽게 말해, **어떤 환경에서든 동일하게 작동하는 독립된 실행 환경(컨테이너)**을 만들고 관리하는 도구라고 할 수 있습니다.

- nginx : Nginx(엔진엑스)는 현재 웹 서버 및 리버스 프록시 시장에서 가장 널리 사용되는 소프트웨어 중 하나입니다.
  고성능과 높은 동시 접속 처리 능력을 제공하도록 설계되었으며, 특히 대규모 트래픽을 처리하는 환경에서 강력한 성능을 발휘합니다.
